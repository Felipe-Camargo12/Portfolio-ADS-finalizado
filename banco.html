Banco de dados - ADS

Banco de dados é uma coleção organizada de informações eletronicamente armazenadas. Ele permite o gerenciamento eficiente e a recuperação dos dados, garantindo integridade e segurança.

MER - Modelo Entidade Relacionamento

Durante as primeiras aulas, aprendemos sobre o Modelo Entidade-Relacionamento (MER), que é uma representação dos requisitos que serão transformados em um modelo de banco de dados. O MER descreve as entidades envolvidas, os atributos que as caracterizam e os relacionamentos existentes entre elas.

O MER permite visualizar de forma clara e abstrata as interações entre os elementos do mundo real e auxilia na criação eficiente e adequada do banco de dados. É uma etapa essencial para a modelagem e implementação do sistema de banco de dados.

DER - Diagrama Entidade Relacionamento

O Diagrama Entidade-Relacionamento (DER) é uma representação visual que deriva do Modelo Entidade-Relacionamento (MER). Ele mapeia entidades, atributos e relacionamentos, facilitando a construção das tabelas do banco de dados. O DER define colunas, chaves primárias e estrangeiras, eliminando redundâncias e garantindo a integridade dos dados. É uma etapa essencial para a construção eficiente do banco de dados.

Modelo Físico

No contexto do aprendizado do PostgreSQL, nos dedicamos à construção do modelo físico do banco de dados. Essa etapa envolve a definição das estruturas de armazenamento, como tabelas, índices e restrições de integridade. É fundamental estabelecer essas estruturas para garantir a organização e a integridade dos dados dentro do banco de dados.
Também exploramos a linguagem SQL (Structured Query Language), que é a linguagem padrão para gerenciamento de dados. Através do SQL, pudemos realizar operações de manipulação e consulta de dados no banco de dados. Isso inclui inserção, atualização, recuperação e exclusão de informações, permitindo-nos interagir de forma eficiente com o banco de dados.
